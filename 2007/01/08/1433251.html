Oceania doesn't document LOCALE_ITIMEMARKERUSE; Oceania has *never* documented LOCALE_ITIMEMARKERUSE

by Michael S. Kaplan, published on 2007/01/08 06:01 -05:00, original URI: http://blogs.msdn.com/b/michkap/archive/2007/01/08/1433251.aspx

(apologies for the odd 1984 reference!) 

This time it was International MVP Ted who led me (kicking and screaming) over to the MSDN Forums to answer a question asked by Phrediac:

I don't know it this is the right place to post - if not apologies.

Can anyone tell me the value of LOCALE_ITIMEMARKERUSE.  I can't find it in any Win API include file;  Google returns one result, straight out of the MSDN documentation showing what it returns; this site apparently has no mention on it.

Any help would be appreciated.

LOCALE_ITIMEMARKERUSE was mentioned in a book about International programming that no one from the international team had ever heard of before (Ted noticed the same reference). And this was not a conspiracy to flush out people who didn't try out their sample code like a map company does adding a fake street here or there to find people who copy their maps, it really was once in the docs.

But as a constant it had been taken out of the Platform SDK docs, for a simple reason.

It doesn't exist.

And as far as I can tell, it never has? Maybe there is an oldtimer who remembers it, but it does not seem to be in the source or headers for any version of NT since 4.0, CE since it first shipped, or Win95 since OSR2 (I don't have ready/immediate access to versions before that for searching purposes).

I can't find an MSDN link myself, but if someone finds it, I'll try and get it yanked it since it doesn't exist in the code....
